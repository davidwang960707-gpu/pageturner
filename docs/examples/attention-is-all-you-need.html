一页读懂

它真正换掉的不是 RNN，而是“顺序处理”的思维。

Transformer 把序列建模拆成两件事：用 attention 让 token 直接互相看见，用 position encoding 告诉模型顺序在哪里。这样训练可以并行，长距离依赖不再被一步步传递稀释。

核心结论只靠 attention 加前馈网络，就能在机器翻译任务上达到当时领先结果，并显著改善训练并行性。
快读判断：读 GPT、BERT、T5、Llama、MoE、长上下文优化之前，这篇是地基。它回答“为什么现代大模型可以堆得这么大”。

公式翻译

Q/K/V 是把“相关性”拆成可计算的检索问题。

一个 token 先投影出 Query、Key、Value。Query 去和所有 Key 计算相似度，再用这个权重加权 Value。多头注意力则是在多个子空间里并行做这件事。

Attention(Q, K, V) = softmax(QKᵀ / √dₖ) V

读者路线图

从这篇论文到今天的大模型，中间有一条清晰演化线。

2017Transformer 提出，证明 attention-only 架构可行。
2018BERT 将 encoder-only 架构用于双向语言理解。

2020+Scaling laws、稀疏专家、长上下文和高效注意力成为扩展主线。
局限与误读

读 Transformer，别只记“Attention 很强”。

误读 1：attention 可以自动理解一切。它提供的是关系建模机制，能力还来自数据、目标函数、规模、优化和后续工程。
误读 2：位置编码只是小细节。没有位置信息，attention 本身对序列顺序不敏感。后来的 RoPE、ALiBi 等长上下文方法都沿着这个问题继续发展。
误读 3：复杂度只有好处。self-attention 的 O(n²) 代价在长上下文下很重，这也是高效注意力、KV cache 和分块策略持续演进的原因。
